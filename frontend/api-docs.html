API Documentation

Current endpoint map used by AI, consultant, and dashboard pages.

Auth

All protected routes require Authorization: Bearer <firebase_id_token>.

Knowledge and AI

GET/api/settings/ai

Read AI runtime settings and available models.

POST/api/settings/ai

Update runtime model and behavior context.

GET/api/knowledge

List uploaded RAG sources for current user.

POST/api/knowledge

Add text knowledge (content, optional source/category).

POST/api/settings/upload

Upload file (PDF/DOCX/XLSX/CSV) and train RAG chunks.

DELETE/api/knowledge/:source

Delete all chunks by source name.

Consultant

GET/api/consultant/settings

Get booking + schedule configuration.

POST/api/consultant/settings

Update consultant settings and weekly schedule.

GET/api/consultant/slots/:date

Get available time slots for YYYY-MM-DD.

POST/api/consultant/bookings

Create booking request (phone, date, timeSlot required).

GET/api/consultant/bookings

List bookings (optional filter by status/date).

PUT/api/consultant/bookings/:id/status

Update booking status: confirmed, rejected, completed.

WhatsApp and System

GET/api/status

WhatsApp connection status.

GET/api/qr

Get QR for WhatsApp link.

POST/api/disconnect

Disconnect WhatsApp session.

GET/api/dashboard/stats

User dashboard totals and recent activity.

GET/api/config/firebase

Public Firebase web config.
